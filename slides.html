The Challenge

GitHub profiles
hide more than
they reveal.

01
Language data is buried across dozens of individual repositories — impossible to see at a glance.
02
Repo lists default to "recently pushed" — top projects by impact are never surfaced first.
03
Profile pages show raw numbers — no visual context for followers, stars, or activity trends.
04
No single view combines profile data, sorted repos, and language breakdown in one place.
02 / 06
The Solution

Hubble surfaces
what matters.

01 Profile card — avatar, bio, location, join date, live stats Profile
02 Repositories sorted by ⭐ stars — language badges, descriptions, update time Repos
03 Live language chart — SVG donut updates as each repo's data loads Live Chart
04 Rate-limit monitor — visible counter, graceful degradation, reset time shown API Safety
03 / 06
Design Philosophy · Dieter Rams, 1980

Good design is
as little design
as possible.

Innovative
Live chart updating as data streams in — new pattern for analytics UIs.
Useful
One search → full picture. No extra steps, no unnecessary pages.
Aesthetic
Single accent colour, Inter typeface, generous whitespace, 8px grid.
Honest
Rate limit visible at all times. Errors explained plainly, never hidden.
Unobtrusive
No gradients, no hero images, no noise. Data is the design.
Minimal
Zero JS dependencies. Every element earns its place on the page.
